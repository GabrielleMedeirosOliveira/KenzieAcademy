“Qualquer tolo pode escrever um código que um computador consiga entender. Bons programadores escrevem código que humanos consigam entender.”

- Martin Fowler


A principal proposta do livro Clean Code é ajudar o leitor a
escrever um bom código. Perceber as diferenças entre um
código ruim e um código bom, bem como transformar um
código ruim em um código bom.


Exemplo de Lista não ordenada
  • Os itens da lista NÃO ordenada não tem numeração;
  • Cada item da lista NÃO ordenada recebem uma marcação;
  • A marcação na frente de cada item é adicionada pelo html;

Se importar com um código é tomar cuidado na hora de:
  • Dar nomes;
  • Criar funções pequenas e especificas;
  • Não ser redundante;
  • Estar sempre atento a erros;
  • Não deixar de refatorar;
  • Utilizar testes limpos para validações.

dicas de como ser um bom programador

  1. Não pare de aprender;
  2. Pense muito antes de começar a escrever seu código;
  3. Escreva códigos simples;
  4. Saiba pedir ajuda;
  5. Compartilhe seu conhecimento.
